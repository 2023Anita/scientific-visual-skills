# Scientific Infographic

用于生成科研信息图：综述图、疾病图谱、教学图、知识总结图、技术路线、分类对比、科研科普图。

核心原则：**先组织信息，再设计视觉**。信息图的目标是讲清楚，不是单纯好看。

## 默认行为

- 如果用户要“生成图”“做图”“出图”“画一张”，直接调用可用的 ChatGPT 生图工具生成最终图像。
- 只有用户明确说“只要提示词 / prompt / 不要生成图”时，才只输出提示词。
- 如果生图工具不可用，输出完整可复制提示词，并说明当前无法直接渲染。
- 不使用真实期刊 logo、真实品牌、真实机构标识。

## 输入槽位

尽量从用户请求中提取；缺失时合理推断，不为非关键字段反复追问。

- 主题 / 标题
- 副标题，可空
- 研究领域：生物医学、材料、纳米、化学、能源、环境、外科、免疫、肿瘤、神经科学等
- 内容类型：机制总结、分类综述、操作流程、疾病图谱、实验路线、手术步骤、对比分析、教学图、技术发展路线
- 核心信息点：3-10 个
- 研究对象：细胞、组织、器官、材料、分子、器件、芯片、病原体、手术结构等
- 结构形式：自动判断、分栏、左右对比、上下流程、中心主体+周围模块、多面板、时间线、矩阵、图谱式布局
- 语言：中文、英文、中英混排、不要文字
- 画幅比例：1:1、4:5、3:4、16:9、5:2、3:2
- 文字密度：少量标签、中等说明、完整标注
- 配色方案
- 禁用元素

## 版式选择

- 分类综述、多个机制并列：分栏模块或矩阵。
- 治疗前后、传统方法 vs 新方法、正常 vs 病理：左右对比。
- 病程、实验流程、手术步骤、技术路线：上下流程或横向流程。
- 一个核心机制展开多个分支：中心主体 + 周围模块。
- 论文图组、综述图、医学图谱：多面板 A/B/C。
- 疾病进展、治疗过程、技术发展：时间线。
- 疾病、系统、器官、材料平台全景梳理：图谱式布局。

不要固定成一种模板。避免普通 PPT、拥挤长图和装饰性元素堆砌。

## 配色系统

根据主题自动选择，不要做廉价蓝紫科技风。

- Nature 系克制学术：灰蓝、雾蓝、冷灰、象牙白；少量珊瑚红、深蓝、金黄。
- Science 系理性机制：蓝灰、中性蓝、青灰；少量砖红、芥黄、深青蓝。
- 柔和粉蓝论文信息图：雾霾蓝、浅蓝灰、灰粉、浅紫、鼠尾草绿。
- 外科图谱 / 医学说明：肉粉、器官红、象牙白、浅蓝、浅灰。
- Cell 系生命科学：深蓝紫、靛蓝、紫红、青蓝、粉橘，少量荧光强调色。

## 生成提示词结构

调用生图工具时，把提示词组织为：

1. 角色：顶级科研信息图设计师、学术图谱设计师、医学/生命科学/材料科学可视化导演。
2. 任务：围绕主题生成“高水平论文综述图质感 + 清晰信息分区 + 学术图谱式表达”的科研信息图。
3. 输入槽位：主题、领域、内容类型、核心信息点、研究对象、语言、比例、文字密度、禁用元素。
4. 版式策略：根据内容自动选择最合适版式。
5. 视觉要求：信息分区、视觉层级、科研逻辑、短标签、留白、高级配色。
6. 禁止项：普通 PPT、电商模板、花哨模板、箭头混乱、文字拥挤、过饱和、真实 logo。
7. 最终标准：一眼看懂主题，模块清楚，层级明确，学术可信。

## 质量门槛

生成前自检：

- 读者是否能快速理解每个模块的作用？
- 标题、模块标题、主体、标签、箭头是否有层级？
- 图形元素是否服务信息表达？
- 是否避免普通 PPT、商业模板、信息拥挤？
- 配色是否有主色、辅色、强调色、背景色分层？

如果不满足，先修正提示词再调用生图工具。
